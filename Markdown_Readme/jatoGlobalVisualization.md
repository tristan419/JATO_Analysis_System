## 0. 一句话目标（North Star）

构建一个可从“全球 -> 大洲 -> 国家 -> 业务明细”逐层钻取的可视化模块，实现 **国家点亮、地球视图、晨昏线、星空背景、时间动态演示**，并与现有 Streamlit 分析体系无缝联动。

---

## 1. 背景与现状

### 1.1 当前现状

- 当前系统主框架：`Streamlit`
- 数据链路：已具备 ETL 与分析数据产出能力
- 当前短板：
  1. 国家层面的可视化表达不够直观
  2. 缺少沉浸式地球与昼夜动态展示
  3. 缺少按大洲的可视化入口与联动路径

### 1.2 关键约束

- 不能中断现有 Streamlit 业务流程
- 地图国家名称与业务国家名称存在别名/拼写差异
- 3D 与高质量动画可能超出纯 Streamlit 原生能力，需要阶段化升级

---

## 2. 项目范围（Scope）

### 2.1 In Scope（本项目包含）

1. 国家标准化映射（raw country -> ISO2/ISO3 -> continent）
2. 世界地图国家点亮（有数据/无数据）
3. 大洲筛选、国家点击联动
4. 地球视图（缩放最小时为全球球体）
5. 晨昏线（按 UTC 动态）
6. 星空背景（可开关）
7. 与现有筛选器联动（时间、品牌、车型等）
8. 文档化、可运维、可回滚

### 2.2 Out of Scope（当前不包含）

- 完整 BI 平台重构
- 彻底迁移到纯 React 单页应用（短期不做）
- 实时流式秒级刷新（后续可扩展）

---

## 3. 成功标准（Success Metrics）

### 3.1 功能指标

- 国家点亮准确率：`>= 99%`
- 大洲筛选联动正确率：`100%`
- 晨昏线位置与时间一致性：`>= 99%`

### 3.2 性能指标

- 首屏加载（缓存命中）：`<= 2.5s`
- 交互响应（常见筛选）：`<= 500ms`
- 地图操作帧率（中档设备）：`>= 30 FPS`（3D 场景）

### 3.3 质量指标

- 关键路径自动化测试覆盖：`>= 80%`
- 线上 P1 缺陷：`0`
- 文档更新及时率：`100%`

## Phase 1：数据标准化底座（1~2 天）

### 目标

把国家字段转换为稳定可视化键值（ISO + Continent）

### 核心任务

1. 建立 `country_alias_map.csv`
2. country -> ISO3 映射与失败回收
3. 产出 `country_presence.parquet`
4. 产出 `country_unmatched.csv` 供人工修复

### 交付物

- 国家标准化脚本
- 匹配率报告（目标 >=99%）

### 验收标准

- 点亮国家与业务统计一致
- 未匹配国家可追溯

---

## Phase 2：2D 全球点亮 MVP（1~2 天）

### 目标

在 Streamlit 中快速上线“可用版本”

### 核心任务

1. 世界地图国家点亮（有数据亮，无数据暗）
2. 大洲筛选（多选）
3. 国家 hover 提示（记录数、更新时间）
4. 与现有筛选联动

### 交付物

- Streamlit 页面新增“Global Visualization”区域
- 交互可用、稳定运行

### 验收标准

- 用户可直接通过地图完成国家层级筛选
- 交互响应符合性能目标

---

## Phase 3：地球模式与视角预设（2~4 天）

### 目标

从“平面图”升级到“地球形态”

### 核心任务

1. 支持 globe/orthographic 视图
2. 大洲预设视角（欧洲/亚洲/美洲/全球）
3. 缩放层级切换策略（平面 <-> 球体）
4. 几何简化与缓存优化

### 交付物

- 地球模式可切换
- 视角预设按钮

### 验收标准

- 视觉与交互明显升级，不卡顿

---

## Phase 4：晨昏线 + 时间轴（2~3 天）

### 目标

引入“地理 + 时间”动态认知

### 核心任务

1. 按 UTC 计算晨昏线（terminator）
2. 时间滑块（按小时/日期）
3. 自动播放（可开关）
4. 昼夜区域视觉层（透明遮罩）

### 交付物

- 晨昏线动态渲染
- 时间控制组件

### 验收标准

- 不同时间点昼夜边界变化正确
- 与地图交互不冲突

---

## Phase 5：星空背景与沉浸体验（2~3 天）

### 核心任务

### 交付物

- 可开关星空背景

### 验收标准

- 视觉增强不影响可读性与性能

## 7. 技术选型论证（结论）

## 7.1 备选方案

| 方案 | 描述                   | 优点               | 缺点             | 结论               |
| ---- | ---------------------- | ------------------ | ---------------- | ------------------ |
| A    | 纯 Streamlit + Plotly  | 快速上线、改动小   | 高级 3D/动画受限 | **MVP 首选** |
| B    | Streamlit + React 组件 | 兼顾交付与高级视觉 | 工程复杂度上升   | **中期推荐** |
| C    | 全量迁移 React         | 可视化能力最强     | 成本高、周期长   | 暂不采用           |

## 7.2 推荐路线

- 短期：`A`（保证交付）
- 中期：升级 `B`（实现晨昏线/星空高质量效果）
- 长期：视业务收益评估是否走 `C`

## 8. 数据模型与接口定义（建议）

## 8.1 数据表：`country_presence`

| 字段         | 类型   | 说明       |
| ------------ | ------ | ---------- |
| country_raw  | string | 原始国家名 |
| country_std  | string | 标准国家名 |
| iso2         | string | ISO2       |
| iso3         | string | ISO3       |
| continent    | string | 大洲       |
| record_count | int    | 记录数     |
| latest_date  | date   | 最新日期   |
| has_data     | bool   | 是否有数据 |

## 8.2 数据质量规则

1. `iso3` 为空 -> 进入 unmatched
2. `record_count < 0` -> 拒绝写入
3. `continent` 不在白名单 -> 告警
4. 输出匹配率与异常明细

## 9. UI 与交互定义

## 9.1 布局

- 左侧：筛选区（时间/大洲/业务维度）
- 中央：地图或地球
- 右侧：国家 KPI 与明细

## 9.2 核心交互

1. 点选大洲 -> 地图聚焦该区域
2. 点选国家 -> 明细联动
3. 时间拖动 -> 晨昏线更新
4. 主题/星空开关 -> 视觉切换

## 10. 测试与验收计划

## 10.1 测试类型

- 单元测试：国家映射、晨昏线计算
- 集成测试：筛选联动、缓存命中
- UI 回归：地图点击、缩放、切换
- 性能测试：首屏、筛选、动画流畅度

## 10.2 UAT 场景（最小集）

1. 仅选择“欧洲”，验证国家高亮范围
2. 切换日期，验证晨昏线位置变化
3. 放大/缩小，验证平面与地球切换稳定
4. 异常国家名输入，验证不阻塞页面
